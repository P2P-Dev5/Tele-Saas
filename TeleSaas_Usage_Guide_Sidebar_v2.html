القائمة الرئيسية

دليل استخدام نظام TeleSaas

نظام داخلي لمتابعة سرعة الرد والتنسيق بين الأقسام من خلال Telegram Bot وGoogle Sheet وChrome Extension، مع واجهة متابعة يومية للإدارة حسب الصلاحيات.

🎯 الهدف

الفكرة العامة

TeleSaas ينظم متابعة الجروبات بين الأقسام، يوثق الرسائل والحالات، يقيس سرعة الرد، ويعرض المؤشرات للإدارة من خلال إضافة المتصفح بدل الاعتماد على المتابعة اليدوية داخل الجروبات.

🧩 المكونات

ثلاثة أجزاء رئيسية

  • TeleSaas Extension: واجهة المتابعة اليومية حسب الصلاحيات.
  • Google Sheet: مصدر البيانات والسجلات والتشغيل.
  • Telegram Bot: يلتقط الرسائل والحالات من الجروبات المطلوبة.
النظام حالياً جاهز للتجربة التشغيلية الداخلية قبل الاعتماد النهائي.

⚙️ التشغيل

طريقة تجهيز النظام

يتم تجهيز النظام مرة واحدة من خلال الشيت، وبعدها تكون المتابعة اليومية من خلال الإضافة.

تركيب الإضافة

يتم تركيب TeleSaas Extension على المتصفح، ثم إدخال رمز الدخول فقط لتحديد الصلاحيات تلقائياً.

إضافة الأقسام

تسجيل الأقسام الرسمية والهاشتاج الرسمي ووقت السماح SLA.

إضافة الموظفين

تسجيل بيانات الموظفين وربط كل موظف بالقسم والدور الوظيفي وTelegram ID.

إضافة الجروبات

تسجيل الجروبات المطلوب متابعتها وإدخال Telegram Chat ID لكل جروب.

ربط الجروبات بالأقسام

تحديد الأقسام الموجودة داخل كل جروب ليتم ربط الرسائل بالمسار الصحيح.

إضافة الموظفين إلى الجروبات

ربط الموظفين بالجروبات وتحديد القسم الذي يمثلونه داخل كل جروب.

إضافة البوت للجروبات

بعد تجهيز البيانات، يتم إضافة البوت للجروبات المطلوبة وتشغيل المتابعة.

🧩 Extension

واجهة المتابعة اليومية

الإضافة هي الواجهة الأساسية للإدارة والموظفين. المستخدم يدخل برمز دخول فقط، والنظام يحدد الصلاحيات تلقائياً.

🔐 صلاحيات

حسب الرمز

  • المدير أو الأدمن: لوحة متابعة عامة وفلاتر.
  • الموظف: البيانات المرتبطة به فقط.
  • المستخدم النهائي لا يحتاج روابط تقنية أو أكواد.
المتابعة اليومية للإدارة تكون من خلال TeleSaas Extension، وليس من شيت الرسوم البيانية.
🤖 Telegram Bot

البوت والجروبات

يتم إضافة البوت فقط إلى الجروبات المطلوب متابعتها. بعد ذلك يلتقط الرسائل التي تحتوي على هاشتاجات التشغيل ويسجلها داخل الشيت.

فتح حالة

رسالة تحتوي على هاشتاج فتح حالة مثل سؤال أو طلب مع تحديد القسم المطلوب.

الرد والمتابعة

يتم احتساب الردود والمتابعات المرتبطة بالحالة أو المسار.

الإغلاق

عند إغلاق الحالة يتم احتساب مدة الحل وهل كانت داخل SLA أم لا.

🚀 الفائدة

قيمة النظام

  • تنظيم متابعة الجروبات بين الأقسام.
  • توثيق الرسائل والحالات بشكل واضح.
  • قياس سرعة الرد حسب القسم أو الجروب أو الموظف.
  • معرفة الحالات المتأخرة أو بدون رد.
📈 متابعة أسرع

Dashboard للإدارة

  • تقليل المتابعة اليدوية داخل الجروبات.
  • توفير واجهة متابعة من خلال الإضافة.
  • تقليل احتمالية ضياع الحالات أو نسيان المتابعة.
  • تجهيز بيانات قابلة للمراجعة والتحليل بشكل أسرع.
📝 ملاحظات

ملاحظات تشغيل مهمة

  • يجب إضافة البوت فقط إلى الجروبات المطلوب متابعتها.
  • يجب تسجيل Chat ID لكل جروب داخل شيت الجروبات.
  • يجب تسجيل Telegram ID لكل موظف داخل شيت الموظفين.
  • يجب ربط الجروبات بالأقسام وربط الموظفين بالجروبات قبل بدء التشغيل الكامل.
  • كل مستخدم يدخل من خلال رمز دخول مخصص حسب صلاحياته.
  • المستخدم النهائي لا يحتاج للوصول إلى الأكواد أو روابط الربط التقنية.
  • النظام حالياً جاهز للتجربة التشغيلية الداخلية قبل الاعتماد النهائي.
